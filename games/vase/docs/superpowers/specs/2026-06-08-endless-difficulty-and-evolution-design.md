# VASE — 무한 난이도 · 진화 계급 확장 설계

작성일: 2026-06-08

## 배경 / 문제

- 게임은 끝이 없다(`game.js` `onWin` → `newGame(level + 1)`). 그러나 난이도는 `core.js`의 `BANDS` 마지막 행 `[14,16,0]`(Lv29+)에서 **천장**을 친다. 이후엔 배치만 랜덤이라 체감 난이도가 안 오른다 → "40레벨까지 너무 쉽다".
- 진화 계급(`EVOS`)도 Lv40 드래곤이 최종이라 그 이후 동기 부여가 없다.
- 초반(Lv1부터 12병)이 튜토리얼 치고 묵직하다.

## 목표

1. **난이도가 무한히 오르되 항상 풀 수 있는** 판을 보장한다.
2. 새로운 난이도 축(숨겨진 층)을 도입한다.
3. 초반에 가벼운 튜토리얼(5병 안팎)을 둔다.
4. 진화 계급을 무한히 잇되, **Lv55에서 서사시(드래곤) → 귀여움(양)으로 반전**하고 이후 더 귀여워진다. 최종은 병아리/숲뭉치 계열.

## 솔버 검증 (선행 측정 — 설계의 토대)

`stress.js`로 측정한 결과(설계 수치는 이 데이터에 종속):

| 구성 | 풀림률(24회내) | 비고 |
|---|---|---|
| `[14,16,0]` 8색 여유8 (현 천장) | 100% / 즉시 | 기준 |
| `[20,22,0]` 8색 여유8 | 100% / 즉시 | 병 22개도 안전 |
| `[18,20,0]` 12색 여유8 | 100% / 즉시 | 12색도 안전 |
| `[16,16,0]` 8색 **여유0** | **0%** | 금지 |
| `[19,20,0]` 12색 **여유4** | 69% / 최대 1.7s | 위험: 폴백 빈발 |
| `[21,22,0]` 12색 **여유4** | 56% / 평균 2.4s | 위험 |

**결론:**
- **여유공간(slack)은 항상 ≥ 8(= N − F = 2)로 고정.** 이 지대에선 병 22 · 색 12까지 100% 즉시 풀림.
- 난이도는 **색↑ · 병↑ · 숨김층**으로 올린다. **slack을 조이지 않는다.**
- `generateLevel`(core.js:236)은 못 풀면 `generateSolvableBoard(numColors, 4)`라는 **쉬운 판으로 폴백**한다. slack을 조이면 이 폴백이 자주 터져 *오히려 쉬운 판*을 주므로, 위 규칙이 폴백 트랩을 회피한다.

## 작업 묶음 (독립 착륙 가능)

원래 불만("너무 쉽다")을 직접 푸는 **① 난이도부터** 착륙. ②, ④는 이후 독립 착륙.

### ① 무한 난이도 곡선 (core.js — 순수 로직, 가장 검증됨)

Lv29 천장 `[14,16,0]` 이후를 **파라미터 공식**으로 잇는다. 모두 검증된 안전지대(slack 8 고정).

`levelConfig(level, numColors, rng)`를 다음과 같이 확장:
- `level`이 `BANDS` 범위 내면 기존 표 사용(단, 초반은 ③ 튜토리얼로 교체).
- 범위를 넘으면 파라미터 분기:

```
step = level - 30                      // Lv30부터
colors = min(12, 8 + floor(step / 8))  // 8→12색 (~Lv62에 12색)
N      = min(22, 16 + floor(step / 6)) // 16→22병 (~Lv66에 22병)
F      = N - 2                         // 색 슬롯 = 병 - 2 → 여유 항상 8
hidden = min(N, 2 + floor(step / 3))   // 숨김 병 수 (묶음②) — 캡 이후의 무한 난이도 담당
```

- 색·병이 ~Lv66에 천장을 치면, 그 뒤로는 **`hidden`만 계속 늘어** 무한 난이도를 담당한다(전부 가려지면 순수 기억 게임).
- **색 팔레트 확장 필요**: `COLORS`는 현재 8개. 12색을 위해 구분 가능한 4색 추가(예: 시안 `#22d3ee`, 핑크 `#ec4899`, 라임 `#84cc16`, 갈색 `#a16207` — 구현 시 색맹 구분성 확인 후 확정). 어두운 색은 `SHADE_HI/SHADE_LO` 보정 추가.
- **`numColors` 스레딩**: 현재 `newGame`이 `NUM_COLORS=8` 상수를 넘긴다. `colorsFor(level)`를 만들어 레벨별 색 수를 계산해 넘기도록 변경.
- `nodeBudget`은 현행 80k 유지(여유8 지대는 즉시 풀림).

**검증:** `stress.js`/`test.js`에 Lv30/50/70/100 구성이 100% 풀리고 par를 돌려주는지(폴백 미발생) 단언 추가.

### ② 숨겨진 층 (점진적 확대)

병 아래층이 `?`로 가려지고, **위를 부어 비워야 드러난다**(클래식 covered 워터소트).

- **생성은 완전정보로 풀리게** 유지 → 숨김은 순수 정보 난이도. 솔버/par 계산에 영향 없음.
- **상태**: 병별 `revealed` 카운트(처음 노출된 칸 수). 가려진 병은 맨 위 1칸만 보이고 그 아래는 `?`. 위가 비워져 새 칸이 top이 되면 `revealed` 증가.
- **undo**: 한 번 본 칸은 계속 보인다(`revealed`는 줄지 않음). 공정성 + 단순성.
- **렌더 변경**(단순 상태 아님): `drawTube`(game.js:201)가 `revealed` 아래 칸을 실제 색 대신 `?`/물음표 텍스처로 마스킹. 가려진 칸의 그라데이션·수면·기포는 중립 회색으로.
- **점진 확대**: 묶음①의 `hidden` 병 수만큼 무작위 병을 "가린 병"으로 지정. Lv30=2병 → 증가 → 고레벨 전체.
- **par/별점 공정성**: par는 완전정보 기준이라 가린 판에서 ≤par(3★)는 사실상 불가. **숨김 병이 있는 레벨은 3★ 기준을 `par×1.5`로 완화**(`starsFor`에 hidden 플래그). 일반 레벨은 그대로.

### ③ 병 개수 확장 + 튜토리얼 (core.js + 그리드)

- **튜토리얼 (Lv1~3, 5병 안팎)**: 초반 밴드를 가볍게. 예: `[3,3,2]`(3색·3병·빈2=5병) → `[4,4,2]` → `[5,5,2]`, 이후 기존 곡선으로 합류. 레벨 *번호*는 그대로 두고 초반 밴드 내용만 가볍게 교체하므로, 절대 레벨 기준인 EVOS 임계값은 유효(아래 ④에서 재확인).
- **16병 초과(~22병) 그리드**: 모바일은 현재 4열·최소 28px(`#board` `index.html:84`, `fitBoard` game.js:107). 18병↑이면 **5열 전환 + 최소폭 하향 + 필요 시 세로 스크롤**. `fitBoard`가 병 수에 따라 열 수/`--w`를 조정하도록 확장. 데스크톱은 8열(760px) 활용.

### ④ 진화 계급 — 서사시 → 귀여움 반전 (game.js EVOS + sprite.svg)

`EVOS`(game.js:16)를 확장. 판정 로직 `evoFor`/`nextEvo`는 그대로(절대 레벨 기준).

|---|---|---|---|
| 1 | 물방울 | (현행) | `g-drop` ✓ |
| 5 | 방울방울 | (현행) | `g-bubbles` ✓ |
| 10 | 열대어 | (현행) | `g-tropical-fish` ✓ |
| 15 | 문어 | (현행) | `g-octopus` ✓ |
| 20 | 돌고래 | (현행) | `g-dolphin` ✓ |
| 25 | 상어 | (현행) | `g-shark-fin` ✓ |
| 30 | 고래 | (현행) | `g-sperm-whale` ✓ |
| 40 | 드래곤 | 서사시 정점 | `g-dragon-head` ✓ |
| **55** | **양** 🐑 | **반전 — 말랑 시작** | 소싱+검증 필요 |

- **아이콘은 콘텐츠 의존성**: `sprite.svg`엔 현재 8개 심볼만 있다. 55+ 항목은 [game-icons.net](https://game-icons.net)에서 **실제 SVG path를 받아 `sprite.svg`에 추가**해야 한다. 위 ID는 미정 — **"소싱+검증"을 별도 작업으로** 분리하고, 검증 전 ID를 `setEvoIcon`에 박지 않는다. 70/85 동물 라인업은 구현 단계에서 확정.
- **EVOS 임계값 재확인**: 튜토리얼(③)은 레벨 번호를 안 밀므로 기존 임계값(5/10/15…) 유효. 신규 임계값(55/70/85/100)이 새 난이도 곡선상 자연스러운지만 확인.

## 기존 유저 마이그레이션

**불필요 — 진화는 저장값이 아니라 매번 계산되는 파생값.** `evoFor(getMaxClear())`가 `vaseMaxClear`만 읽어 아이콘을 정한다(단계 자체는 미저장).

- 결정: **조용히 자동 적용.** Lv57 유저는 다음 접속 시 `evoFor(57)` → 양으로 자동 표시. 변환 작업 없음.
- 진화 토스트(game.js:420)는 실제 클리어 시에만 뜨므로, 복귀자에겐 안내 없이 조용히 바뀐다(의도된 동작).

## 비목표 (YAGNI)

- slack 4 이하의 타이트 구성(폴백 트랩).
- 얼음/잠금 등 특수 병(이번 범위 밖).
- 최종 진화 이후의 별(⭐) 카운트 등 추가 위신 표식(추후 검토).

## 테스트 전략

- `test.js`: 새 곡선의 Lv30/50/70/100 구성이 100% 풀리고 폴백 미발생(par 반환) 단언.
- `test.js`: 숨김 레벨에서 `revealed`가 줄지 않음(undo 후에도), par는 완전정보 기준 불변 단언.
- 수동: 모바일에서 22병 그리드 가독성, 색 12개 구분성, 튜토리얼 5병 체감.

## 착륙 순서

1. **① 난이도 곡선** (불만 직접 해소, 순수 core.js, 가장 검증됨)

3. **④ 진화 계급** (EVOS + 아이콘 소싱)
4. **③ 그리드/튜토리얼**은 ①과 함께(병 수 변경이 맞물림) 또는 직후
